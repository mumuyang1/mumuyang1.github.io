jQuery

jQuery是一个封装了Javascript的的轻量的库。让JavaScript操作DOM（Document Object Model）变得简单。并且解决了很多跨浏览器的兼容问题。

  • 操纵DOM（以面向对象方式描述的文档模型，文档访问方式）
  • 制作动画效果
  • 事件处理
  • Ajax请求

http://www.w3schools.com/jquery/jquery_intro.asp

Bootstrap

Bootstrap是HTML，CSS和JS的响应式设计的Web框架。

源码是基于CSS预处理脚本Less 和Sass开发的.如果不想写CSS的话就可以使用他提供的一些样式和
组件。

gulp

gulp 是基于 node 实现 Web 前端自动化开发的构建工具。
基于流的自动化构建工具

gulp的优点

      通过代码优于配置的策略，gulp让简单的任务简单，复杂的任务好管理

  • 利用Nodejs流的威力，快速构建项目并减少频繁的IO操作
  • gulp有严格的插件确保指南。保证开发的插件高品质。
  • 通过最少的API就可以掌握它，构建工作尽在掌握，如图一系列的流管道。
gulp的API

什么是src和pipe?

我们先看一个unix命令：

cat gulpfile.js | wc -l

这是两个独立的命令，cat gulpfile.js用来获取gulpfile.js的文件内容，wc -l用来统计文件中的行数，他们中间用“|”相连。把cat gulpfile.js的输出作为wc -l的输入。这是再常见不过的unix命令

gulp中吸取了这样的想法，上面gulp命令可以想作：

‘spec/google.spec.js’ | jasmine()

Node.js

      Node.js 是基于Chrome V8引擎的 JS 的运行环境。基于C++使用事件驱动，非阻塞式I/O的模型。包管理器是npm。
单核单线程。
事件驱动机制是Node.js通过内部单线程高效率地维护事件循环队列来实现的，没有多线程的资源占用和上下文切换

http://www.runoob.com/nodejs/nodejs-http-server.html

npm start

如果没有配置npm start node 会默认运行 node server.js

npm

node的包管理工具

bower

web的包管理器

几个常用的命令

npm install             下载在package.json 中配置的依赖
npm list                列出已经安装的模块
npm show express        显示模块的详情
npm update              升级当前目录下项目的所有模块
npm update expresss     指定升级
npm uninstall express   删除已下载的模块
安装
npm install -g bower

bower的升级安装全部依赖于bower

npm 和 bower的区别

与NPM最大的区别在于，NPM主要运用于Node.js项目的内部依赖包管理，安装的模块位于项目根目录下的
node_modules文件夹内。而Bower大部分情况下用于前端开发，对于CSS/JS/模板等内容进行依赖管理，
依赖的下载目录结构可以自定义。
