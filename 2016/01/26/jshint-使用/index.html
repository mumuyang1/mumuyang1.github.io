jshint 使用

jshint

jshint是一个JavaScript代码的检查工具。

比如你希望确保你的代码中不会出现定义了确没有用的变量，都使用单引号，该写分号的地方都写上分号等等等等。jshint就是这样一些工具，根据你的配置检查你的js代码，找出不符合要求的地方。

安装

npm install -g jshint

安装后比如你想检查你的demo.js文件

// demo.js
var a = 1;
var b = 2
var c = a + b;

运行

➜  jshint demo.js
demo.js: line 2, col 10, Missing semicolon.
1 error

配置

新建文件配置文件.jshintrc，在这个文件里可以写上你自己的配置。

一些配置选项

curly
使用循环或者条件语句必须使用花括号包起来

newcap
构造函数首字母大写

noarg
禁止使用 arguments.caller 和 arguments.callee，未来会被弃用， ECMAScript 5 禁止使用 arguments.callee

multistr
检查多行字符串

eqeqeq
如果为真，比较时必须使用!== 或者 === 而不是!= 或者 ==

quotmark
值可以为"double"或者"single"表示只能使用双引号或者单引号

latedef
如果值为true的话应该先声明函数或变量等，然后再使用。

strict
值为真时，表示code运行在严格模式下。需要加'use strict';

globals
这个选项可以用于指定一些我们在代码中非正常声明的全局变量。如果值为true的时候允许读写这个变量，
值为false的话是只读的。

更多请戳官网 http://jshint.com/docs/options/
